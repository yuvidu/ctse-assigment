# Booking Service API Documentation

This document provides a comprehensive overview of the REST API endpoints available in the Booking Service.

## Base URL
The service runs on `http://localhost:8080` (default port) with the base path for bookings at `/booking`.
The frontend is expected at `http://localhost:5173`.

---

## Endpoints

### 1. Create a New Booking
- **URL:** `/booking`
- **Method:** `POST`
- **Description:** Creates a new booking record in the system.
- **Request Body:** [Bookingmodel](file:///d:/ctse-assigment/Booking-Backend/booking-service/src/main/java/com/yuvidu/booking_service/model/Bookingmodel.java#8-77) (JSON)
- **Response:** [Bookingmodel](file:///d:/ctse-assigment/Booking-Backend/booking-service/src/main/java/com/yuvidu/booking_service/model/Bookingmodel.java#8-77) (The created booking object)

### 2. Get All Bookings
- **URL:** `/booking`
- **Method:** `GET`
- **Description:** Retrieves a list of all bookings.
- **Response:** `List<Bookingmodel>`

### 3. Get Booking by ID
- **URL:** `/booking/{id}`
- **Method:** `GET`
- **Description:** Retrieves a specific booking by its unique identifier.
- **Path Parameters:**
    - `id` (String): The ID of the booking.
- **Response:** [Bookingmodel](file:///d:/ctse-assigment/Booking-Backend/booking-service/src/main/java/com/yuvidu/booking_service/model/Bookingmodel.java#8-77)

### 4. Get Bookings by User ID
- **URL:** `/booking/user/{userId}`
- **Method:** `GET`
- **Description:** Retrieves all bookings associated with a specific user.
- **Path Parameters:**
    - `userId` (String): The ID of the user.
- **Response:** `List<Bookingmodel>`

### 5. Update Booking Status
- **URL:** `/booking/{id}/status`
- **Method:** `PUT`
- **Description:** Updates the status of an existing booking.
- **Path Parameters:**
    - `id` (String): The ID of the booking.
- **Query Parameters:**
    - `status` (String): The new status (e.g., "CONFIRMED", "PENDING").
- **Response:** [Bookingmodel](file:///d:/ctse-assigment/Booking-Backend/booking-service/src/main/java/com/yuvidu/booking_service/model/Bookingmodel.java#8-77) (The updated booking object)

### 6. Cancel a Booking
- **URL:** `/booking/{id}/cancel`
- **Method:** `PUT`
- **Description:** Cancels a specific booking.
- **Path Parameters:**
    - `id` (String): The ID of the booking.
- **Response:** [Bookingmodel](file:///d:/ctse-assigment/Booking-Backend/booking-service/src/main/java/com/yuvidu/booking_service/model/Bookingmodel.java#8-77) (The updated booking object with "CANCELLED" status)

## Data Models

### Bookingmodel
The primary data structure used for requests and responses.

| Field | Type | Description |
| :--- | :--- | :--- |
| `id` | `String` | Unique identifier for the booking (Auto-generated by MongoDB). |
| `userId` | `String` | ID of the user who made the booking. |
| `showId` | `String` | ID of the movie show/event being booked. |
| `seats` | `List<String>` | List of seat identifiers (e.g., ["A1", "A2"]). |
| `status` | `String` | Current status of the booking (e.g., "PENDING", "CONFIRMED", "CANCELLED"). |
| `paymentId` | `String` | Reference ID for the payment transaction. |
| `createdAt` | `LocalDateTime` | Timestamp when the booking was created. |
